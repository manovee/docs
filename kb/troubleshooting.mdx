# Troubleshooting guide

Use this guide when something in Tamlr does not behave as expected.

## I cannot sign in

Try these steps:

1. Confirm you are using the correct email.
2. Use the password reset option from the sign-in page.
3. If using Google sign-in, confirm the same Google email is attached to your account.
4. Ask an admin to confirm your invite or account status.

If the problem continues, ask an admin to confirm that your account is active and invited to the right workspace.

## I signed in but cannot see a company

You may not belong to a company yet.

1. Check for pending invites during onboarding.
2. Ask a workspace admin to invite you.
3. Create a company if your plan allows it.

If you expected to see an existing workspace, ask a workspace admin to resend the invite or confirm your role.

## The dashboard has no data

The dashboard needs completed prompt runs.

1. Go to **AI Search**.
2. Create or activate prompts.
3. Run AI tests.
4. Wait for responses and sentiment analysis.
5. Return to **Dashboard**.

If the dashboard still looks empty after successful tests, confirm that you are viewing the correct company workspace and date range.

## Prompt limit reached

Your plan controls how many active prompts the workspace can use.

Options:

- Deactivate prompts you no longer monitor.
- Delete unused prompts if history is not needed.
- Ask an admin to assign a higher plan.

## AI tests fail or stay incomplete

Possible causes:

- A provider is disabled or misconfigured.
- The selected model is unavailable.
- The AI provider rate-limited the request.
- A search or sentiment provider failed.
- A required platform setting is missing or outdated.

Ask a platform admin to check **Admin > Usage Analytics** and **Admin > Configuration**.

## Source domains are missing

Source Domains depends on citations in stored AI responses.

1. Run AI tests for prompts that ask research or recommendation-style questions.
2. Open prompt detail and confirm responses include citations.
3. Refresh Source Domains.

## Competitor suggestions look wrong

Competitor suggestions are based on recent AI answers and company context.

1. Dismiss irrelevant suggestions.
2. Track important competitors manually.
3. Run more prompts that reflect real buying questions.
4. Refresh from AI responses.

## Products are not discovered from a website

Try these steps:

1. Use the main company website or a product/category URL.
2. Confirm the website is public.
3. Add products manually if the website is sparse or blocked.

## CMS connection test fails

Check:

- Site URL is correct.
- WordPress application password is active.
- Webflow token has access to the right site and collection.
- HubSpot private app token has the required blog/content scopes.
- The external platform is reachable.

## Admin page is missing

Admin appears only for superadmin accounts. Ask an existing platform admin to confirm your role.

## What to include when asking for help

The most useful support details are:

- Page where the problem happened.
- Company workspace name.
- Prompt title or product name involved.
- Time of the attempted action.
- Any visible error message.
- Whether the action was a settings change, AI run, CMS publish, or admin action.

Platform admins can cross-check AI usage, provider settings, subscription plans, and user roles.
